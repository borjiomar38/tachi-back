# Nayovi Creator Platform Screenshot Rights Packet

This packet is maintained by the SEO distribution agent for owner-reviewed creator-platform, publisher, localization, app-review, and directory conversations.

Use this as a source-of-truth checklist before any public screenshot, demo clip, app-testing report, Product Hunt gallery, YouTube demo, Reddit post, LinkedIn post, or partner pilot mentions Nayovi processing manga, manhwa, manhua, comics, or webtoon pages.

## Purpose

Nayovi should be easy for reviewers and partners to evaluate without implying that any publisher, platform, creator, or catalog has granted processing rights. Every public visual should prove the Android OCR/translation workflow while keeping content rights, creator credit, privacy, and platform rules clear.

## Allowed Sample Types

| Sample type | Public screenshot allowed | Notes |
| --- | --- | --- |
| Owner-created test pages | yes, after owner approval | Best default for public demos and directory galleries. |
| Public-domain pages | yes, after source and license review | Keep citation/source notes with the asset reference. |
| Official sample pages | only if the sample terms allow this use | Do not crop away terms, credits, or ownership context to imply broader permission. |
| Creator-provided approved samples | only with written approval and scope | Store only the non-secret approval reference in docs, never private contracts or email contents. |
| User-owned private pages | no public screenshot by default | Use only for private support/debugging unless the owner explicitly approves publication. |
| Publisher/platform catalog pages | no, unless permission is explicit | Never imply Nayovi can process WEBTOON, K MANGA, GlobalComix, or any other catalog content without approval. |

## Public Visual Checklist

Before an image or video is used publicly, confirm:

- The sample source is approved for the exact public use.
- The screenshot does not expose private user data, tokens, emails, device identifiers, license keys, or support conversations.
- The image keeps enough context to avoid implying official platform endorsement.
- The caption says Nayovi does not host or distribute chapters.
- The caption says the workflow is for owned, public-domain, official-sample, or permission-approved content.
- The translated output is described as OCR/AI-assisted draft support that still needs human review before public use.
- The asset reference is stored as a filename, URL, or approved library reference only; no private credentials or contracts are copied into docs.

## Suggested Public Caption

`Nayovi Android OCR/AI translation workflow shown with an approved sample. Nayovi does not host or distribute chapters; use it only with owned content, public-domain material, official samples, or content you have permission to process. AI-assisted output should be reviewed before sharing.`

## Partner Pilot Boundaries

For creator platforms, publishers, localization studios, and editorial teams:

- Start with a no-link note and ask whether an approved-sample OCR QA checklist would be useful.
- Do not ask for a backlink, catalog access, public endorsement, or permission to process platform content.
- If a pilot is invited, ask the owner to approve sample scope, screenshot/publication scope, review-code access, and follow-up contact path.
- Keep all legal, financial, commercial, and rights decisions with the owner.

## Owner Confirmation Needed

The owner still needs to provide:

- Approved public screenshot set.
- Public/private asset library reference.
- Package name and signing-certificate fingerprint for APK trust packets.
- Android developer verification/package registration status.
- Written sample-use scope for any creator-provided or platform-provided pages.
