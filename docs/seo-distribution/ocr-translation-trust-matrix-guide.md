# Android OCR Translation Trust Matrix for Nayovi Distribution Prospects

## Purpose

This document is a practical, low-friction asset for reviewers, directories, and partner conversations.
It consolidates how the app should be presented without overclaiming and without linking to unpublished
verification states.

## One-line product truth

Nayovi is an Android APK workflow for manga, manhwa, and manhua OCR + translation support with free
trial onboarding and paid monthly token plans. It does not host chapters.

## Trust matrix (external-fit language only)

| Audience | Allowed claim | Avoided claim |
| --- | --- | --- |
| Android review or API directory editors | Official APK source + pricing/supported plans + support path + screenshot policy | "Play Store verified", "unverified chapter processing", "open source" unless owner-confirmed |
| Technical publishers | Hosted OCR workflow + trial-to-paid split + responsible-use boundary | "Best in class", "unlimited free trials", hidden affiliate-style framing |
| Creator/localization teams | Approved-sample only, creator-owned or permissioned flow, glossary/consistency focus | "Can process entire publisher catalogs", unauthorized sample assumptions |
| Investors / enterprise diligence | Package ownership intent, review policy, no-mirror language, source links | "Complete launch metrics" or private code/signing values |

## Reviewer-ready decision gates

- Always send `no-link-first` context when target rule risk is unclear.
- Share direct links only after target form or contact path confirms source-link handling.
- Never mix APK trust context with unsupported claims about verification status.
- Keep package/signing facts to pending/publicly confirmed language until owner-approved confirmation arrives.

## Use in SEO and distribution work

1. Use as a reusable link target in `platform-drafts.md`.
2. Use as the first fallback when a prospect asks for quality criteria.
3. Use as a citation bridge before any launch/submit action.

## Open follow-up

Owner review required for any public claim touching:

- package name
- signing fingerprint
- developer verification status
- APK metadata scope and screenshot rights
